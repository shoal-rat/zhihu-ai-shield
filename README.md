# 知乎 AI 清净器

把知乎里的争吵、钓鱼、阴阳怪气和低信息密度作者挡在屏幕外。

这是一个 Manifest V3 浏览器扩展。它会在你刷知乎时自动识别当前内容作者，抓取当前卡片和作者近作样本，交给本机 OpenAI-compatible 小模型判断；命中后把内容改写成一个安静的占位块，也可以直接隐藏、折叠或模糊。

## 它解决什么

知乎的拉黑通常发生在你已经被恶心到之后。这个扩展的目标是把这一步提前：

- 自动扫描知乎首页、问题页、回答页、专栏页里的作者卡片
- 优先读取作者近 3 条内容作为判断样本
- 用本机 LLM 判断是否属于低质量争吵型作者
- 对命中内容本地屏蔽，并缓存判断结果
- 支持白名单、黑名单、阈值、缓存时间和展示方式调整
- 模型不可用时只记录错误并放行，不做关键词兜底硬判

## 重要边界

当前版本做的是“本地屏蔽”，不是替你调用知乎官方拉黑接口。原因很直接：知乎内部接口可能变化，也可能造成误封、误操作或账号风险。本地屏蔽已经能覆盖主要阅读场景，并且更可控。

扩展不会绑定任何第三方服务。默认模型接口是本机地址：

```text
http://127.0.0.1:11434/v1/chat/completions
```

你可以接 Ollama、LM Studio、vLLM 等本机兼容接口。默认不预置远程模型域名。

游客兼容只处理未登录时知乎已经公开加载到页面里的内容，不伪造登录态，不绕过服务端限制，也不集成任何“免登录绕过”第三方代码。

## 安装

1. 下载或克隆本仓库。
2. 打开 Chrome/Edge 的扩展管理页。
3. 开启“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目根目录，也就是包含 `manifest.json` 的目录。
6. 点击扩展图标，进入设置页，确认本机 LLM 配置。

## 本机模型示例

如果你使用 Ollama，可以先准备一个小模型：

```bash
ollama pull qwen2.5:0.5b
ollama serve
```

然后在扩展设置中填：

```text
Endpoint: http://127.0.0.1:11434/v1/chat/completions
Model: qwen2.5:0.5b
```

模型名不固定，填你本机实际存在的 OpenAI-compatible 模型即可。

如果扩展里测试模型提示 `403`，通常是 Ollama 没放行浏览器扩展 Origin。Windows 上可以设置：

```powershell
setx OLLAMA_ORIGINS "chrome-extension://*,http://localhost:*,http://127.0.0.1:*"
```

然后重启 Ollama。

## 怎么判断

扩展会把样本整理成一段非常克制的提示词，要求模型只看文本行为：

- 人身攻击
- 引战钓鱼
- 阴阳怪气
- 谣言口吻
- 低信息密度
- 情绪化争吵

它明确要求模型不要根据性别、年龄、地域、民族、国籍、职业、疾病、宗教、政治身份等身份属性判断。这个项目只做个人阅读过滤，不做群体标签。

## 设置建议

- 阈值：默认 `72%`，如果误伤多就调高到 `80%`
- 最低置信度：默认 `42%`，小模型不稳定时可以调高
- 近作篇数：默认 `3`，速度和效果比较平衡
- 命中后处理：默认“改写成占位块”，方便临时展开和纠错
- 缓存小时数：默认 `168`，也就是一周

## 文件结构

```text
.
├── manifest.json
├── src/
│   ├── background.js          # 队列、缓存、近作抓取、本机 LLM 调用
│   ├── content/               # 知乎页面扫描与屏蔽 UI
│   ├── options/               # 设置页
│   ├── popup/                 # 扩展弹窗
│   └── shared/                # 分类提示词、默认配置、知乎解析工具
├── tests/                     # Node 内置测试
├── scripts/check-manifest.mjs # manifest 路径与权限校验
└── docs/                      # 中文说明文档
```

## 开发检查

本项目没有打包步骤，直接加载源码即可。提交前可以跑：

```bash
node scripts/check-manifest.mjs
node --test tests/*.test.mjs
```

如果 PowerShell 禁止 `npm.ps1`，直接用上面的 `node` 命令即可。

## 隐私

默认情况下，扩展只在浏览器本地存储设置、名单、缓存结果和最近判断记录。样本文本只会发送到你配置的本机模型接口。更详细的边界见 [docs/隐私说明.md](docs/%E9%9A%90%E7%A7%81%E8%AF%B4%E6%98%8E.md)。
